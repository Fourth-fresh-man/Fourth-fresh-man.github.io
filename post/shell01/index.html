Shell脚本 基础 - 01

2020-12-12

shell是我在实习时候经常会用的语言，一般在项目提供部署，领导要求一些好看的报表时候用的比较多，目前我用shell大概3个月，只会用一些很普通的命令，其实在网上查阅资料的时候就发现了shell许多nb的用法，特此一一记录下来。

shell脚本的定义

shell脚本默认是用户使用linux的桥梁，底层使用C语言实现。
bash默认是shell脚本执行的解释器,每次编写shell时，我都会将其加在头文件，告诉系统这是一个shell脚本，这样创建时无需指定.sh格式。

#!/bin/bash

shell脚本执行

我每次创建完后，都会

chmod +x xxx

这样，可以用./xxx方式来执行脚本，也可以赋予更高权限
这里还有个小技巧，在根目录下创建bin，把脚本放在bin里，可以全局快速使用。

shell变量权限设置和删除

readonly 变量名将会使变量变成只读变量，也就意味着如果后面想要更改此变量，将会报错
unset 变量名会删除该变量，变量将永远消失，但他不能删掉只读变量(readonly)

shell字符串

shell字符串尽量使用双引号，依据我在项目实施的经验，这样可以避免很多语义上的问题，还有就是引用变量的时候，用花括号${}会使代码结构更清晰

# 拼接字符串，双引号和单引号区别
your_name="runoob"
# 使用双引号拼接
greeting="hello, "$your_name" !"
greeting_1="hello, ${your_name} !"
echo $greeting  $greeting_1
# 使用单引号拼接
greeting_2='hello, '$your_name' !'
greeting_3='hello, ${your_name} !'
echo $greeting_2  $greeting_3

输出结果为：
hello, runoob ! hello, runoob !
hello, runoob ! hello, ${your_name} !

获取字符串长度，#将会把后面的元素作为一个可读字符串对象，获取其长度

string="abcd"
echo ${#string}

结果为：6

在数组上就体现为

name=(
    usa
    cn
    uk
)
for country in name;
do
    echo ${#country}
done

将每个数组元素当成可读对象，分别取长度，用echo $取值

# 提取子字符串
str="this is a string"
echo ${string:0:3} # 输出this

# 查找子字符串，查找字符i或o的位置，输出最先出现的那个
str="this is a string"
echo `expr index "$str" io` # 输出3
